Get Started in 60 Seconds

From installation to your first query. No complex setup, no infrastructure to manage.

1

Install Hektor

$ pip install hektor

Or build from source for maximum performance and customization.

2

Create Your First Collection

import hektor

client = hektor.Client()
collection = client.create_collection(
    name="documents",
    dimension=512,
    metric="cosine"
)
3

Add Data & Search

# Add documents
collection.add(
    texts=["AI and machine learning", "Deep neural networks"],
    metadata=[{"source": "paper1"}, {"source": "paper2"}]
)

# Semantic search
results = collection.search(
    query="artificial intelligence",
    top_k=5
)

for result in results:
    print(f"{result.text} (score: {result.score})")

Frequently Asked Questions

How is Hektor different from Pinecone or Weaviate?
Hektor is an embedded database written in C++ with explicit SIMD optimizations. We're not a cloud service or a managed offering. You run Hektor in your own infrastructure with zero network latency and deterministic performance. Our focus is on raw speed and memory efficiency.
Can I use Hektor in production?
Yes! Hektor v3.0 is production-ready with comprehensive documentation, extensive test coverage, distributed system support, and observability features including Prometheus metrics. We have users running Hektor in production for semantic search, RAG pipelines, and recommendation systems.
What embedding models does Hektor support?
Hektor comes with ONNX Runtime integration for local embedding generation. We ship with MiniLM-L6-v2 for text and CLIP ViT-B/32 for images. You can also use external embeddings from OpenAI, Cohere, or any custom model. We're model-agnostic—just provide the vectors.
Does Hektor support hybrid search?
Absolutely. Hektor v3.0 includes full BM25 full-text search with 5 fusion algorithms (RRF, Weighted Sum, CombSUM, CombMNZ, Borda Count). You get the best of both worlds: semantic understanding from vectors and precise keyword matching from BM25.
How do I scale Hektor horizontally?
Hektor v3.0 includes distributed system support with replication (async/sync/semi-sync), sharding (hash/range/consistent), and automatic service discovery. Deploy on Kubernetes with our Helm charts, or use Docker Compose for simpler setups. We also provide gRPC clients for multi-language support.
Is there a hosted version of Hektor?
Currently, Hektor is self-hosted only. This gives you complete control over your data and infrastructure. However, we're exploring managed offerings for teams that prefer a hands-off approach. Join our Discord to stay updated.
